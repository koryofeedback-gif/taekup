# PARTIE 2 : SYSTÈME DE GAMIFICATION "DOJANG RIVALS"

## 2.1 Concept Fondamental

**Dojang Rivals** est un système original de gamification conçu spécifiquement pour les arts martiaux, transformant l'entraînement en une expérience compétitive et engageante pour les élèves de tous âges.

## 2.2 Architecture des Défis

### 2.2.1 Catégorisation Tripartite

| Catégorie | Description | Exemples |
|-----------|-------------|----------|
| **Power (Puissance)** | Défis de force et d'explosivité | Nombre de pompes, hauteur de saut |
| **Technique** | Défis de précision et de forme | Qualité des formes, précision des coups |
| **Flexibility (Souplesse)** | Défis d'amplitude et d'étirement | Grand écart, souplesse dorsale |

### 2.2.2 Les 19 Défis Originaux

1. Push-up Challenge (Puissance)
2. Kick Height Challenge (Technique)
3. Plank Endurance (Puissance)
4. Form Precision (Technique)
5. Split Progress (Souplesse)
6. Speed Kicks (Puissance)
7. Balance Hold (Technique)
8. Jumping Kicks (Puissance)
9. Stretching Routine (Souplesse)
10. Breaking Power (Puissance)
11. Poomsae Memorization (Technique)
12. Core Strength (Puissance)
13. Flexibility Score (Souplesse)
14. Sparring Points (Technique)
15. Cardio Endurance (Puissance)
16. Stance Accuracy (Technique)
17. Hip Flexibility (Souplesse)
18. Reaction Speed (Puissance)
19. Movement Flow (Technique)

## 2.3 Système de Points d'Expérience (XP)

**Mécanisme original :** Attribution de points XP pour chaque défi complété, avec bonus progressifs.

| Action | Points XP |
|--------|-----------|
| Défi complété | 10-50 XP (selon difficulté) |
| Streak quotidienne | +10 XP bonus |
| Défi Mystery Box | +25 XP bonus |
| Team Battle victoire | +100 XP |
| Parent-Child Challenge | +75 XP |

## 2.4 Fonctionnalités Innovantes

### 2.4.1 Team Battles (Batailles d'Équipes)
Système de compétition par équipes au sein du dojo, favorisant la cohésion et la motivation collective.

### 2.4.2 Parent-Child Challenges (Défis Parent-Enfant)
Mécanisme d'engagement familial permettant aux parents de participer aux défis avec leurs enfants.

### 2.4.3 Belt-Tier Matchmaking (Appariement par Ceinture)
Algorithme d'équité assurant que les compétitions se font entre élèves de niveaux comparables.

### 2.4.4 Daily Streak Bonuses (Bonus de Séries Quotidiennes)
Système de récompense pour l'assiduité quotidienne, encourageant l'engagement régulier.

### 2.4.5 Mystery Challenge Box (Boîte Mystère)
Mécanisme de découverte aléatoire de défis avec récompenses bonus, ajoutant un élément de surprise.

### 2.4.6 Challenge Inbox (Boîte de Réception des Défis)
Interface centralisée pour la réception et la gestion des notifications de défis.

### 2.4.7 Coach Challenge Builder (Constructeur de Défis)
Outil permettant aux coachs de créer des défis personnalisés avec :
- Sélection de catégorie
- Niveaux de difficulté
- Types de mesure (temps, répétitions, distance, score)
- Liens vidéo optionnels
- Templates prédéfinis

### 2.4.8 Rivals XP Dashboard
Tableau de bord coach affichant les niveaux d'engagement et statistiques XP détaillées de chaque élève.

# PARTIE 3 : SYSTÈMES D'INTELLIGENCE ARTIFICIELLE

## 3.1 TaekBot - Assistant IA

### 3.1.1 Concept
Assistant conversationnel spécialisé dans les arts martiaux, offrant des réponses contextuelles et personnalisées.

### 3.1.2 Persona et Comportement
```
TaekBot est un assistant expert en arts martiaux qui :
- Répond dans la langue configurée du club
- Adapte ses conseils au type d'art martial pratiqué
- Fournit des informations techniques précises
- Maintient un ton professionnel et encourageant
- Connaît le contexte du club (nom, style, règles)
```

### 3.1.3 Paramètres de Personnalisation
- Nom du club
- Type d'art martial (Taekwondo, Karaté, Judo, etc.)
- Langue de communication
- Règles spécifiques du dojo

## 3.2 AI Class Planner - Planificateur de Cours IA

### 3.2.1 Fonctionnalité
Génération automatique de plans de cours structurés basés sur :
- Niveau de ceinture des participants
- Zone de focus (technique, cardio, formes, combat)
- Durée du cours (30, 45, 60, 90 minutes)
- Nombre d'élèves
- Langue de sortie

### 3.2.2 Structure de Sortie
Plan de cours détaillé incluant :
- Échauffement
- Exercices techniques
- Travail spécifique
- Conditionnement physique
- Retour au calme
- Temps alloué par section

## 3.3 Welcome Email Generator - Générateur d'Emails de Bienvenue

Création automatique d'emails de bienvenue personnalisés pour les nouveaux élèves/familles.

### 3.3.1 Paramètres
- Nom du club
- Nom de l'élève
- Nom du parent
- Type d'art martial
- Langue

## 3.4 Black Belt Time Machine - Prédicteur de Progression

### 3.4.1 Concept Original
Algorithme de prédiction calculant la date estimée d'obtention de la ceinture noire basé sur :
- Ceinture actuelle
- Fréquence d'entraînement
- Durée moyenne entre promotions
- Calendrier des vacances et jours fériés
- Historique de l'élève

### 3.4.2 Innovation
Intégration du calendrier des vacances scolaires françaises et des jours fériés pour une précision accrue des prédictions.

## 3.5 Architecture IA Duale

### 3.5.1 Stratégie de Routage
| Tâche | Modèle | Justification |
|-------|--------|---------------|
| TaekBot (conversations complexes) | GPT-4o | Qualité supérieure |
| AI Class Planner | GPT-4o | Créativité et structure |
| Tâches routinières | Google Gemini | Coût optimisé |

### 3.5.2 Système de Fallback
Logique de repli automatique en cas d'indisponibilité d'un modèle, assurant la continuité du service.

# PARTIE 6 : ARCHITECTURE TECHNIQUE

## 6.1 Stack Technologique

| Composant | Technologie |
|-----------|-------------|
| Frontend | React 18, TypeScript, Vite 5 |
| Styling | Tailwind CSS |
| Backend | Express.js, TypeScript |
| Base de données | PostgreSQL (Neon) |
| ORM | Drizzle ORM |
| Paiements | Stripe |
| Déploiement | Vercel |
| IA | OpenAI GPT-4o, Google Gemini |

## 6.2 Architecture Serveur Duale

**Innovation :** Consolidation automatique dev/prod

| Environnement | Architecture |
|---------------|-------------|
| Développement | Vite Dev Server + Express API Server |
| Production | Express Server unique (API + Static) |

## 6.3 Pattern de Proxy IA Sécurisé

Toutes les clés API (OpenAI, Gemini, Stripe) sont stockées côté serveur uniquement. Le frontend utilise des endpoints proxy sécurisés.

## 6.4 Système de Protection par Mot de Passe

Mécanisme de contrôle d'accès pré-production avec :
- Vérification côté serveur
- Stockage de session navigateur
- Activation/désactivation par variable d'environnement

## 6.5 Optimisations de Performance

- Chargement parallèle des API
- Cache des produits Stripe (TTL 5 minutes)
- Index de base de données optimisés
- Support vérifié : 5,000+ élèves, 10,000+ défis quotidiens
